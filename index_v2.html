Category Definitions — 3-Filter Logic
F1 Impressions · F2 (ROAS or Cost/NCP) · F3 Cost/FTEWV
Evaluation thresholds
F1 — Min Impressions
50,000
Must pass before evaluation is considered valid.
F2 — ROAS OR Cost/NCP
ROAS≥ 3.2 ×
Cost/NCP≤ ₹ 525
Passes when either ROAS or Cost/NCP clears its bar.
F3 — Cost / FTEWV ✓
≤ ₹ 12
First-time engaged-view-purchase cost.
Categorisation matrix

Each unique ad is evaluated against the four filters. A ✓ means the filter must pass for that category; a — means it isn't checked; a ~ means either state is fine (the row wins on other columns). Categories are matched top-down — the first matching row wins.

◆ F3 (Cost/FTEWV) is a universal quality metric — it's evaluated for every ad and acts as the upgrade gate across categories (F1-only → P1; F1+F3 → P0; F1+F2 → Winner; F1+F2+F3 → Incremental Winner). P2 stays P2 whether F3 passes or not — F2's presence with no F1 is the deciding signal.
⏳ 14-day evaluation buffer — ads within their first 14 days from ad_created that hit no filter drop into Result Awaited instead of Discarded. Once the ad crosses day 14 without lighting any filter, it falls through to Discarded as before. Categories that DO match a filter (Winner / P0 / P1 / P2) are unaffected by the buffer.
Category F1 F2 (ROAS or Cost/NCP) F3 (Cost/FTEWV) Action
Incremental Winner ✓ ✓ ✓ Scale aggressively. Top-tier proven creative.
Winner ✓ ✓ ~ Scale. F3 close-second; iterate on cost-per-FTEWV.
P0 analysis ✓ — ✓ Impressions met + cheap FTEWV. Iteration target.
P1 analysis ✓ — — Hit impressions, but F2 (ROAS / Cost-NCP) and F3 (Cost-FTEWV) both miss.
P2 analysis — ✓ ~ Strong F2 signal (F3 either state) but still building impressions.
Result Awaited — — — No filter passed yet, but the ad is still inside its 14-day evaluation window from ad_created. Verdict is deferred.
Discarded — — — Cut spend or rework creative — no signal after 14 days of delivery.
✓ required — not required ~ either state OK F2 = ROAS OR Cost/NCP (either passes → F2 passes)

Thresholds are set in refresh_summary_table.py. To tune them, edit the constants in the script and rerun the full update; the funnel flags f1_pass – f4_pass on each ad in ae_table_view get recomputed on every refresh.
