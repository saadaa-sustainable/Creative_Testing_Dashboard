Product mix donut

  • Collection page — name contains CLP
  • Product page — name contains PDP · VRP · "product"

Number of new ads created per week, bucketed by their ad_created date (Monday-anchored). Header total = sum of all weekly counts in the current view. Use this to spot bursts or droughts in creative production.

Creative mix

Creative mix donut
Distribution of ads by asset-code marker in ad_name:
  • IFAD — Influencer / UGC partnership ad
  • GAD — Graphic ad (static image / poster)
  • VID — Video ad (matches VID as a whole word)
  • UGC — User-generated-content style
  • ADB — Ad-batch / re-uploaded creative
  • BR — Brand-led ad (matches BR_)
  • BST — Bestseller-focused ad
  • Other — no marker recognised

Category Definitions — 4-Filter Logic
F1 Impressions · F2 ROAS · F3 Cost/NCP · F4 Cost/FTEWV
Evaluation thresholds
F1 — Min Impressions
50,000
Must pass before evaluation is considered valid.
F2 — Min ROAS
3.2 ×
Conv. Value ÷ Spend over the date range.

≤ ₹ 525

F4 — Cost / FTEWV ✓

Categorisation algorithm

Each unique ad is evaluated against a four-filter matrix. F1 is mandatory for any Winner-class bucket. Incremental Winner passes all four. Winner clears F1–F3 (F4 not required). Ads that pass only F1 + F4 land in Priority (P0 iteration target). Ads passing F1 alone fall to Analyze 1; ads passing F2 (but not F1 — still building impressions) go to Analyze 2. Everything else is Discarded.

CategoryConditionAction
Incremental Winner F1 ✓ AND F2 ✓ AND F3 ✓ AND F4 ✓ Scale aggressively. Top-tier proven creative.
Winner F1 ✓ AND F2 ✓ AND F3 ✓ Scale. F4 close-second; iterate on cost-per-FTEWV.
Priority F1 ✓ AND F4 ✓ Impressions met + cheap FTEWV. P0 iteration target.
Analyze 1 F1 ✓ only Hit impressions, but ROAS / Cost-NCP / Cost-FTEWV all miss.
Analyze 2 F2 ✓ (F1 not yet) Strong ROAS signal but still building impressions.
Discarded else (no filter passes) Cut spend or rework creative — no signal.

Thresholds are set in refresh_summary_table.py. To tune them, edit the constants in the script and rerun the full update; the funnel flags f1_pass – f4_pass on each ad in ae_table_view get recomputed on every refresh.
