Creative mix donut
Distribution of ads by asset-code marker in ad_name:

Category Definitions — 4-Filter Logic
F1 Impressions · F2 ROAS · F3 Cost/NCP · F4 Cost/FTEWV
Evaluation thresholds
F1 — Min Impressions
50,000
Must pass before evaluation is considered valid.
F2 — Min ROAS
3.2 ×
Conv. Value ÷ Spend over the date range.
F3 — Cost / NCP
≤ ₹ 525
Σ Spend ÷ Σ NCP count over the date range.
F4 — Cost / FTEWV ✓
≤ ₹ 12
First-time engaged-view-purchase cost.
Categorisation algorithm

Each unique ad is evaluated against a four-filter matrix. F1 is mandatory for any Winner-class bucket. Incremental Winner = F1 + (F2 or F3) + F4. Winner = F1 + (F2 or F3) — F4 not required. Ads that pass only F1 + F4 land in P0 analysis (iteration target). Ads passing F1 alone fall to P1 analysis; ads passing F2 (but not F1 — still building impressions) go to P2 analysis. Everything else is Discarded.

CategoryConditionAction
Incremental Winner F1 ✓ AND (F2 ✓ OR F3 ✓) AND F4 ✓ Scale aggressively. Top-tier proven creative.
Winner F1 ✓ AND (F2 ✓ OR F3 ✓) Scale. F4 close-second; iterate on cost-per-FTEWV.
P0 analysis F1 ✓ AND F4 ✓ Impressions met + cheap FTEWV. Iteration target.
P1 analysis F1 ✓ only Hit impressions, but ROAS / Cost-NCP / Cost-FTEWV all miss.
P2 analysis F2 ✓ (F1 not yet) Strong ROAS signal but still building impressions.
Discarded else (no filter passes) Cut spend or rework creative — no signal.

Thresholds are set in refresh_summary_table.py. To tune them, edit the constants in the script and rerun the full update; the funnel flags f1_pass – f4_pass on each ad in ae_table_view get recomputed on every refresh.
